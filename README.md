# Azure Horizon HRM 系统操作说明书

本说明书针对系统中涉及「个人目标申请」、「团队任务下发」以及「赏金榜提案发布与审核」的核心操作流进行规范。

## 一、任务填写的通用原则 (Markdown 沉浸式编辑)

目前，系统中所有的任务表单（包含个人目标、部门下发、提议招募等）均已切换为 **Markdown 富文本编辑器** 模式：

1. **模板自动填充**：
   - 打开新增或编辑窗口时，系统会自动填充 `【目标 S】`、`【指标 M】`、`【方案 A】`、`【相关 R】`、`【时限 T】` 的格式模板。
   - 用户只需在相应的标签节点下方，直接输入相关细节描述即可。
2. **图文与排版支持**：
   - 可以在文本中自由使用加粗、列表、表格，以及粘贴 Markdown 格式的代码和图片链接。
3. **内容无损流转**：
   - 无论经过何种审批或打回重审环节，您在编辑器中填写的所有排版文本均会**完整保留**，不会被切割或丢失。

---

## 二、PDCA 执行时间的强制校验规约

为保证任务执行的闭环性和可追溯性，所有的核心节点均受 **PDCA 强制校验** 控制。PDCA 的四个时间节点支持**精确到分钟**的双向选择与保存。

### 1. 个人发起 / 主管下发场景
- **草稿豁免**：在填写一半时，您可以随时点击「存为草稿」离开。存草稿时**不会**强制校验时间的完整性。
- **强制约束**：当您点击「发起申请」进行最终提交时，如果下方的 **Plan（计划）、Do（执行）、Check（检查）、Act（复盘）** 四个日历输入框存在空缺，系统将会拦截并弹窗提示补全。

### 2. 赏金提案 / 人事一键开启场景
- **提案阶段**：基层员工在发起“新提案”时，可选择性提交。
- **高管圈定**：高管/总办在审批提案时，只需核对方向与预算。
- **人事落地 (关键点)**：
  - 当人事收到审批通过的新提案并在「公司大盘」中点击 **分配一键启动** 时，弹出的是该任务的终极确认窗口。
  - **此时 PDCA 时间规划区将处于强制打通的编辑状态**。人事或项目统筹必须依照规划补充完整所有的 PDCA 精确时间。若未填写，将无法成功派发和点燃该项赏金任务。

## 三、各业务板块流转逻辑梳理

### (一) 公司大盘（赏金榜池）
- **发起建议**：任何员工均可发起建议。
- **多级审批**：提案需经主管初筛 → 人事风控核查 → 高管拍板敲定。
- **揭榜认领**：变成招募贴后，全员可自由意向认领（点击 R 意向申请执行，点击 A 意向申请监督）。
- **确认分配**：招募截止后，分配者需决定最终入选成员，修改正文补充最终确定的要求，**填写完整的 PDCA 时间节点**，点击「分配一键启动」后彻底生效，淘汰其余落选者。

### (二) 个人/团队目标看板 (Smart 看板)
- **目标直达**：点击个人/团队页面中的「创建目标」，直接生成新卡片进入周期池。
- **直接完成机制**：
  - 若为普通周期目标（且指派人与审批人为同一人时），当您觉得进度达到且材料充足，可直接触发 **提前完结任务**，并填写结案原因直通 STAR 评级广场，免去了繁杂的中间手续。

### (三) STAR 验收获评 (极简评级表)
- 达到时间节点或被强制完结的任务，都会进入“待阅卷”或者“已评分”的 STAR 广场。
- 相关人员在此可针对任务结果，提交打分、结案汇报资料以及复用资产总结，实现从发起至结案的全链路知识归档。
