## 功能概览

**能力大盘**是一套完整的人才素质评估管理系统，涵盖：

- 构建自定义能力评估模型（多维度、可配置权重与满分）
- 发起团队能力评估，走「员工自评 → 主管打分」两阶段闭环流程
- AI 辅助出卷，创建并派发能力测评试卷
- 实时追踪评估进展，可视化状态流转
- 团队能力短板预警，识别人才缺口
- 维护预设能力库，保障组织评估体系一致性

## 模块详解

### 1. 待办测评

> **谁会看到**：有待完成测评试卷的员工

此 Tab 会在标签名旁显示红色数字角标，提示待完成数量。

**页面内容**：
- 以卡片列表展示所有待完成的测评试卷
- 每张卡片显示试卷名称和简要说明

**操作步骤**：

1. 点击卡片上的 **「立即答题跑出雷达图」** 按钮
2. 弹出答题窗口，显示试卷标题、作答须知
3. 逐题作答（单选/多选），页面实时显示「共 X 题，已答 Y 题」
4. 全部作答后点击 **「提交答卷」**
5. 系统弹出确认框：「确定要交卷吗？交卷后将立即出分且无法修改。」
6. 点击 **「确认交卷」**，即刻显示得分
7. 若该试卷关联了能力维度，分数将自动同步至对应的能力评估记录

> **注意**：交卷后无法修改答案，但仍可查看历史作答记录。

### 2. 团队评估进展

> **谁会看到**：所有用户（管理员看全部，员工仅看自己相关的）

**页面功能**：

#### 搜索与筛选
- **搜索框**：支持按姓名、模型名、下发人搜索
- **部门筛选**：下拉选择部门，可选「全部部门」
- **分页控制**：每页 9 条，支持翻页

#### 评估卡片
每张卡片展示一条评估记录，包含：

| 信息 | 说明 |
|------|------|
| 状态徽标 | 🔵 待自评 / 🟡 待主管评分 / 🟢 已完成 |
| 被评人 | 员工姓名（大字显示） |
| 元信息 | 使用的模型名、所属部门、下发人 |
| 进度条 | 三节点可视化：员工自评 → 主管打分 → 结果归档 |

#### 操作按钮（卡片底部）
- **待自评**状态 → 显示「立即自评」（仅本人可见）
- **待主管评分**状态 → 显示「主管打分」（仅指定主管可见）
- **已完成**状态 → 显示「查看报告」

点击按钮会打开 [评估评分弹窗](#8-评估评分流程)。

### 3. 能力短板与预警

> **谁会看到**：管理员 / 主管

基于全部已完成的评估数据，自动聚合分析团队能力短板。

**页面内容**：
- 标题：「团队能力短板预警」
- 说明：基于已完成评估，对比预留基线（目标分）后计算出的平均技能掌握度

**预警卡片**：

每个能力维度生成一张卡片，样式按状态区分：

| 状态 | 判定规则 | 样式 |
|------|---------|------|
| ⚠️ 人才短缺 | 考核均分 < 基线目标分 | 红色左边框 + 红色背景 |
| ✅ 稳定 | 考核均分 ≥ 基线目标分 | 绿色左边框 + 绿色背景 |

卡片信息：
- **能力分类**（如：通用基石、核心基础、专业技能等）
- **能力名称**
- **考核均分**：所有已完成评估中该维度的平均分
- **基线/最高分**：达标目标分 / 历史最高分

> 此页面为只读信息面板，帮助管理者快速定位需加强培训或招聘的技能领域。

### 4. 能力模型配置

> **谁会看到**：管理员 / 主管

**用途**：创建和管理评估时使用的能力模型。每个模型由多个考核维度组成。

#### 新建模型

1. 点击右上角 **「新增模型」**
2. 弹出编辑窗口，填写：

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 模型名称 | ✅ | 如「高级研发序列能力模型」 |
| 适用部门 | — | 如「业务部 / 技术部」 |
| 岗位说明 | — | 描述适用的岗位职责、用途或考核背景 |

3. 添加能力维度（通常 4–6 项），有两种方式：

   **方式 A：从能力库导入**
   - 点击 **「从能力库导入」** 按钮
   - 弹出预设能力卡片列表，点击即可导入
   - 自动填充维度名称、分类、说明

   **方式 B：手动追加**
   - 点击 **「手动追加维度」** 按钮
   - 手动填写维度名称、满分、达标分、权重

4. 每个维度需配置：

| 参数 | 默认值 | 说明 |
|------|:------:|------|
| 维度名称 | — | 如「技术视野」「沟通协作」 |
| 满分 | 10 | 该维度评分上限 |
| 达标分 | 6 | 目标/基线分，低于此分触发预警 |
| 权重 | 1 | 在综合评分中的权重占比 |

5. 点击 **「保存配置」**

#### 编辑 / 删除模型
- 鼠标悬停卡片右上角显示编辑（✏️）和删除（🗑️）图标
- 已被评估引用的模型**不可删除**

#### 校验规则
- 模型名称不能为空
- 至少包含 1 个能力维度
- 维度名称不能为空
- 所有维度的满分必须大于 0

### 5. 预设能力库（字典）

> **谁会看到**：管理员 / 主管

**用途**：维护一套组织级的通用能力字典，保障跨模型评估体系一致性。能力库中的条目可在创建模型时快速导入。

**页面为表格形式**，字段如下：

| 列名 | 说明 |
|------|------|
| 类别 | 能力分类（下拉选择） |
| 能力名称 | 如「前端架构设计」 |
| 能力解读 | 描述达标所需的行为表现 |
| 默认满分 | 导入模型时的默认上限分 |
| 操作 | 编辑 / 删除 |

**类别选项**：
- 通用基石
- 核心基础
- 专业技能
- 管理领导
- 业务理解
- 其他拓展

#### 新增入库
1. 点击右上角 **「新增入库」**
2. 选择类别、填写名称（必填）、满分基准、能力解读
3. 点击 **「保存入库」**

#### 编辑 / 删除
- 点击行末的编辑或删除图标
- 删除不会影响已引用该能力的模型

### 6. 考卷题库与发卷

> **谁会看到**：管理员 / 主管

**用途**：创建测评试卷、AI 智能出卷、向员工派发测试、查看答卷结果。

#### 新建试卷

1. 点击 **「新建套卷」**
2. 填写基本信息：

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 试卷名称 | ✅ | 如「前端高级工程师定级测试」 |
| 试卷说明 | — | 测试目的、注意事项等 |
| 关联维度 | — | 下拉选择能力库中的维度。关联后，员工提交试卷会自动折算分数写入该维度评估 |

3. 添加题目（两种方式）：

**方式 A：AI 一键出卷**
- 点击 **「AI一键出卷」** 按钮
- 填写信息辅助 AI 生成：

| 字段 | 说明 |
|------|------|
| 测试目的 | 如「评估候选人的系统架构设计能力」 |
| 关联岗位 | 如「高级Golang工程师」 |
| 拟生成题数 | 默认 5 题，最多 20 题 |
| 岗位职能要求 | 任职资格或软硬技能要求 |
| 参考材料 | 可手动输入、导入网页链接、上传文档（PDF/TXT/MD/CSV） |

- 点击 **「一键生成」**，AI 自动生成题目并插入试卷
- 管理员可在生成后手动调整任何题目

**方式 B：手动加题**
- 点击 **「手动加题」**
- 配置每道题：

| 配置项 | 说明 |
|--------|------|
| 题型 | 单选题 / 多选题 |
| 题干 | 题目内容 |
| 分值 | 该题分数 |
| 选项 | 至少 2 个选项，点击字母按钮标记正确答案 |

4. 点击 **「保存试卷」**

#### 派发测试

1. 在试卷卡片上点击 **「派发测试」**
2. 弹出人员选择面板：
   - 搜索框可按姓名筛选
   - 勾选需要派发的员工（支持多选）
3. 点击 **「确认派发 (N人)」**
4. 被选中的员工将在「待办测评」Tab 中看到该试卷

#### 查看结果

点击试卷卡片上的 **「查看结果」**，弹出统计面板：

| 统计项 | 说明 |
|--------|------|
| 总派发人数 | 派发了多少人 |
| 已完成人数 / 完成率 | 已提交答卷的比例 |
| 平均得分 | 所有已完成答卷的平均分 |

下方表格列出每位员工的详细情况：姓名、部门、状态（待作答/已完成）、得分、提交时间。

### 7. 发起评估

> **入口**：页面右上角 **「发起评估」** 按钮（仅管理员/主管可见）

**操作步骤**：

1. 点击 **「发起评估」**
2. **选择考核标准**：从下拉列表选择已创建的能力模型（必选）
3. **选择评估对象**：
   - 搜索框支持按姓名、部门筛选
   - 勾选需要评估的员工（支持全选当前搜索结果）
   - 右上角显示「已选 N 人」
4. 点击 **「立即下发 N 人」**

**系统自动处理**：
- 为每位选中的员工创建评估记录（状态：待自评）
- 按模型的所有维度生成评分记录
- 发起人自动成为该评估的主管评分人
- 若员工已有同模型的进行中评估，自动跳过并提示

> **提示**：下发后，被评估人将在「团队评估进展」中看到自己的待自评任务。

### 8. 评估评分流程

评估采用 **两阶段闭环** 流程：

```
员工自评 ──→ 主管打分 ──→ 结果归档（自动）
```

#### 阶段一：员工自评

1. 员工在「团队评估进展」中看到状态为「待自评」的卡片
2. 点击 **「立即自评」**
3. 弹出评估表，显示所有能力维度
4. 对每个维度：
   - 输入自评分数（0 ~ 满分，步长 0.5）
   - 可填写评价说明（记录该维度表现的客观评价或事实依据）
5. 点击 **「提交评分问卷」**
6. 状态自动流转为「待主管评分」

#### 阶段二：主管打分

1. 主管在「团队评估进展」中看到状态为「待主管评分」的卡片
2. 点击 **「主管打分」**
3. 弹出评估表，显示：
   - 左侧：员工自评分数（只读）
   - 右侧：主管打分输入框
4. 主管对每个维度打分并可添加评语
5. 点击 **「提交评分问卷」**
6. 状态自动流转为「已完成」

#### 阶段三：查看报告

评估完成后，点击 **「查看报告」** 可查看：
- 雷达图式的能力概览（各维度进度条）
- 每个维度的员工自评分 vs 主管评分对比
- 双方的文字评语
- 三节点进度指示器（全部亮起）

## 典型工作流

### 工作流 1：完整评估周期

```
① 管理员录入预设能力库
    ↓
② 管理员基于能力库创建评估模型（选择维度、配置满分/权重）
    ↓
③ 管理员发起评估 → 系统批量生成评估记录
    ↓
④ 员工完成自评打分
    ↓
⑤ 主管复核并完成打分
    ↓
⑥ 评估闭环 → 生成雷达图
    ↓
⑦ 管理员查看「能力短板预警」，识别培训/招聘需求
```

### 工作流 2：测评驱动的能力评估

```
① 管理员创建试卷（手动或AI出卷）
    ↓
② 关联试卷到某个能力维度（可选）
    ↓
③ 派发试卷给指定员工
    ↓
④ 员工在「待办测评」完成答题
    ↓
⑤ 系统自动评分
    ↓
⑥ 若关联了能力维度，分数自动同步至对应评估
    ↓
⑦ 管理员在「查看结果」中查看完成率和得分分布
```

### 工作流 3：AI 辅助出卷

```
① 管理员进入试卷编辑 → 点击「AI一键出卷」
    ↓
② 填写测试目的、关联岗位、职能要求
    ↓
③ 可选：上传参考文档或导入网页内容
    ↓
④ 点击「一键生成」→ AI 自动生成题目
    ↓
⑤ 管理员审阅/微调题目后保存
    ↓
⑥ 试卷即可派发
```

## 常见问题

### Q：删除能力模型会影响已有评估吗？
A：不会。已被评估引用的模型无法删除，系统会拦截并提示。

### Q：删除能力库条目会影响已引用该条目的模型吗？
A：不会。能力库条目删除后，已使用该条目的模型中对应维度不受影响。

### Q：员工提交答卷后可以修改吗？
A：不可以。交卷后立即出分，答案不可更改，但可以查看历史答题记录。

### Q：同一员工可以被同一模型重复评估吗？
A：同一时间不可以。若员工已有该模型的进行中评估，下发时会自动跳过。上一轮评估完成后才能发起新一轮。

### Q：试卷关联能力维度后，分数是如何同步的？
A：员工提交试卷后，系统将试卷得分自动折算并写入该员工对应能力维度的评估记录中。

### Q：AI 出卷支持哪些参考材料格式？
A：支持手动输入文本、导入网页 URL、上传 PDF / TXT / Markdown / CSV 文件。

### Q：能力短板预警的判定规则是什么？
A：当某能力维度的**全员考核均分**低于该维度设定的**达标分（基线）**时，标记为「⚠️ 人才短缺」红色预警。
